## Текущий статус: ✅ Реализовано

**Ключевые улучшения по сравнению с предыдущим планом:**
- Реальное время: watchdog вместо расписания Task Scheduler
- Безопасность: валидация CSV перед импортом
- Производительность: батчи по 1000 строк (было 500)
- Надежность: задержка 5 сек для завершения записи файла 1С

## Реализованное решение

### Компоненты

1. **1С выгрузка** — CSV файл в сетевой папке:
   - Путь: `\\kz-srv1.lan.dns-shop.kz\kazakhstan\Отдел функционального развития\B2B\РМКО`
   - Файл: `РМКО_выгрузка.csv`

2. **Python-скрипт мониторинга** (`sync_daemon.py`):
   - Использует `watchdog` для реального мониторинга файловой системы
   - При обнаружении изменения файла — ждет 5 сек (чтобы файл полностью записался)
   - Валидирует структуру CSV перед импортом
   - Импортирует данные в Turso батчами по 1000 строк
   - Сохраняет состояние последней синхронизации
   - Режимы работы:
     - `monitor` — непрерывный мониторинг (по умолчанию)
     - `once` — однократная синхронизация (для тестирования)

3. **Установка** (`install_sync_monitor.bat`):
   - Создает задачу в Windows Task Scheduler
   - Запуск при старте системы
   - Автоматическая установка зависимости `watchdog`

### Файлы

| Файл | Описание |
|------|----------|
| `sync_daemon.py` | Мониторинг + импорт с валидацией |
| `install_sync_monitor.bat` | Установка как системной службы |
| `sync.log` | Лог работы синхронизации |
| `.sync_state.json` | Состояние последней синхронизации |

### Характеристики

- **Реакция на изменения:** < 10 секунд (5 сек задержка + обработка)
- **Производительность:** батчи по 1000 строк
- **Время синхронизации:** ожидается < 2-3 минут для типичных объемов
- **Безопасность данных:** валидация CSV перед импортом, защита от некорректных данных
- **Надежность:** отслеживание состояния, логирование всех операций

## Использование

### Установка

1. Запустите `install_sync_monitor.bat` от администратора
2. Скрипт автоматически:
   - Проверит Python
   - Установит `watchdog` если нужно
   - Создаст задачу в планировщике

### Ручное управление

```bash
# Однократная синхронизация (тестирование)
python sync_daemon.py once

# Непрерывный мониторинг (в консоли)
python sync_daemon.py monitor
```

### Управление службой

```bash
# Просмотр задачи
taskschd.msc

# Удаление задачи
schtasks /delete /tn "RMKO_AutoSync_Monitor" /f
```

## Валидация данных

Перед импортом проверяется:

1. **Файл существует и доступен для чтения**
2. **Файл не пустой** — есть заголовки и данные
3. **Обязательные колонки:**
   - `НомерДокумента`
   - `ДатаСоздания`
4. **Кодировка** — корректное чтение CSV

При ошибке валидации импорт отменяется для защиты базы данных.

## Производительность

### Оптимизации

- **Батчи по 1000 строк** — минимум HTTP round-trips к Turso
- **Реальное время** — нет лишних проверок по расписанию
- **Задержка 5 сек** — файл успевает записаться полностью 1С

### Ожидаемое время

- Малый файл (< 1000 строк): < 30 секунд
- Средний файл (1000-5000 строк): 1-2 минуты
- Большой файл (5000-10000 строк): 2-3 минуты
- Очень большой файл (> 10000 строк): 3-5 минут

## Логирование

Все операции записываются в `sync.log`:

- Запуск/остановка мониторинга
- Обнаружение изменений файла
- Результаты валидации
- Статистика импорта (новые/обновленные/без изменений/пропущено)
- Ошибки с полным stack trace

## Устранение неполадок

### Файл не обнаруживается

- Проверьте путь к сетевой папке
- Убедитесь что есть доступ к сети
- Проверьте права доступа к папке

### Валидация не проходит

- Откройте CSV файл и проверьте структуру
- Убедитесь что кодировка соответствует настройке
- Проверьте наличие обязательных колонок

### Синхронизация медленная

- Проверьте скорость интернета
- Проверьте размер файла
- Посмотрите логи для выявления узких мест
