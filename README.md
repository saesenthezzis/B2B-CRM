# Athena One (РМКО) — рабочее место корпоративного отдела

Мини-CRM для работы со сделками В2В: очереди «что делать сегодня», правка этапов
и комментариев, контроль ошибок закрытия, аналитика для руководителя.
Логика статусов перенесена из листа «ИнструкцияДляMain» рабочей Google-таблицы.

Стек: **Python (Flask)** + **SQLite** + **ванильный JS/CSS** (дизайн-система Aether, glassmorphism).


## Аутентификация

- **Регистрация** по email с кодом подтверждения (SMTP настраивается в `.env`).
- **Белый список** email'ов: если в таблице `allowed_emails` есть записи — регистрация только для них.
- При первом входе пользователь обязан сменить временный пароль «1».
- Сессии на основе cookie (`session`), `SECRET_KEY` из `.env`.

## Обновление данных из 1С

1. Скачайте свежий файл Google-таблицы: *Файл → Скачать → Microsoft Excel*.
2. Замените файл `Рабочее место Корпоративного отдела .xlsx` в папке «В2В проект».
3. Нажмите кнопку **«⟳ Обновить из 1С»** в шапке РМКО
   (или запустите «Обновить данные из 1С.bat»).

При обновлении правки менеджеров **не затираются**: данные 1С обновляются,
изменённые сделки получают флаг UPD, новые — NEW.

## Как работать менеджеру

- Выберите себя в списке «Менеджер» (без этого правки не сохраняются) — «Город»
  подставится автоматически по зонам ответственности; у специалистов с
  несколькими городами выбирается «Зона менеджера». Выбор запоминается в браузере.
- Идите по очередям слева направо: **Новые → Требуют действия → Закрытые → Без продажи → Все**.
  В подсказке у ошибочных сделок написана конкретная причина.
- Меняйте «Этап сделки», «След. шаг», даты, причины, товар и прочее прямо в таблице.
  **Изменения буферизируются** — при правке поля строка подсвечивается и справа
  появляются кнопки ✓ (сохранить) / ✕ (отменить). Сделка перемещается в нужную
  группу **только после подтверждения** — это позволяет заполнить все поля строки
  до того, как она «улетит» в другую очередь.
- Фильтры: Период (Сегодня / День / Неделя / Месяц / Год / Вручную), Статус,
  Этап, «Только мои», текстовый поиск по контрагенту/№/телефону.
- Правила закрытия: **Закрыто** = ✔товар в наличии + дата; **Удалён** = причина
  удаления + дата (инициатива компании); **Не состоялась** = причина отказа + дата
  (инициатива клиента). Нарушения попадают в «Требуют действия / Ошибки».
- Выданные в 1С накладные закрываются автоматически: этап «Закрыто», ✔товар,
  проверка и след. шаг — «Закрыто автоматически».
- 💬 — WhatsApp по телефонам клиента; 🕘 — история изменений сделки; клик по «№» — копирование.

## Отчёт

Вкладка **«Отчёт»** — дашборд с аналитикой, учитывающий те же фильтры что и таблица:

- **KPI-карточки**: активные сделки, под риском, выдать товар, просроченные контакты, ошибки заполнения, выдано, потеряно.
- **Графики** (Chart.js): воронка по этапам, потери по причинам, динамика по неделям, города (актив/выдано).
- **Таблица менеджеров**: сделок, сумма, активные, под риском, просрочено, ошибки, выдано.

## Настройки

Вкладка **«Настройки»** — управление зонами ответственности (B2B специалист + город):
добавление и удаление записей. Список специалистов определяет выпадающий список «Менеджер»
в шапке и автоматический выбор города.

| Файл | Назначение |
|------|------------|
| `app.py` | Веб-сервер Flask, порт 8000, API, авторизация |
| `core.py` | Импорт xlsx → SQLite, бизнес-логика статусов и правил |

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/meta` | Справочники, города, этапы, текущий пользователь |
| GET | `/api/deals` | Все сделки с вычисленными полями |
| PATCH | `/api/deal/<key>` | Обновить поля сделки (заголовок `X-User`) |
| GET | `/api/history/<key>` | История изменений сделки |
| POST | `/api/import` | Запустить импорт из xlsx |
| POST | `/api/specialists` | Добавить специалиста |
| DELETE | `/api/specialists/<id>` | Удалить специалиста |
| POST | `/api/auth/send-code` | Отправить код на email |
| POST | `/api/auth/register` | Регистрация по коду |
| POST | `/api/auth/login` | Вход по логину/паролю |
| POST | `/api/auth/change-password` | Смена пароля |
| POST | `/api/auth/logout` | Выход |

## Сроки и статусы

Сроки считаются в **рабочих днях** (сб/вс не учитываются).
Текущий статус из 1С: Выдан (РН проведена, резерв снят) / Резерв / Удалён.
Подсказки:
- счёт не оплачен ≥3 раб.дн. — «связаться»; 
- резерв ≥2 раб.дн. — «контроль»; 
- оплачено без товара — «ждём поступление»; 
- оплачено + товар — «выдать клиенту».
